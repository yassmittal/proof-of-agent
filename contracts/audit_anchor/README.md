# audit_anchor

A small Sui Move package that records one agent run on-chain, permanently.

## What it does

After a run is stored on Walrus, the SDK calls `anchor_run` with the agent's identity, the
covenant hash, and the head of the receipt hash chain — plus a reference to the actual Walrus
`Blob` object that holds the run.

The interesting part: the contract doesn't take your word for the blob. It reads the blob's
content ID and object ID **directly from the on-chain `Blob`** (`blob.blob_id()`), so the anchor
is provably tied to a real Walrus blob, not just a string someone passed in. It then **freezes**
the resulting `AuditAnchor` object, which makes it immutable — the publisher can't go back and
edit history. Every anchor also emits a `RunAnchored` event so the run is easy to discover.

The read-only accessors at the bottom (`chain_root`, `walrus_blob_id`, …) are there so other Move
code could build on an anchor later.

## Files

- `sources/audit_anchor.move` — the module (one entry function, one object, one event).
- `Move.toml` — package manifest. Depends on the real Walrus contracts so it can use the `Blob`
  type; the testnet addresses are pinned under `[dep-replacements.testnet]`.
- `Move.lock` / `Published.toml` — generated by the Move toolchain (resolved deps and the
  published package address). Don't edit these by hand.
- `build/` — compiler output. Generated, gitignore-able, safe to delete and rebuild.

## Working with it

```bash
sui move build
sui move test
sui client publish        # prints the package ID -> put it in ../../.env as AUDIT_ANCHOR_PACKAGE_ID
```
